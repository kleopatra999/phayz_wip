Package maintainer changes

Every Frugalware package has a nominated maintainer, someone who is responsible for keeping it current. As people join and leave the team, some packages change maintainer. A new team member might particularly like Ruby applications and so take over maintainership of Ruby-based packages. Those which don't have an active maintainer are considered 'orphaned' but don't worry because even orphaned packages are still updated if needed. In recent weeks, developers centuri0 and Pingax have taken over maintainership of some orphaned packages. Does anyone else want to take a lonely but loving package into their home? Anyone?

GNOME 3 is now available

As announced on FIMXE by FIXME, GNOME 3 is now available in the Frugalware 'current' repository. This is arguably the most significant GNOME release because it completely changes the way users interact with the desktop. Some people are excited about the release and others are worried by such a big change. Try it for yourself and see what you think.

Move to /tmpfs instead of FIXME

On FIXME_date Frugalware developer VMiklos announced that he wanted all packages which used FIXME modified so that they used FIXME instead. He started this change for his own packages and asked all other package maintainers to do the same. In only FIMXE days the change for all packages was complete, which goes to show just how quickly the development team can respond when required. "But what is /tmpfs?" I hear you ask. Well...a simple explanation is that it's a file system in which files are stored temporarily. Tips and Tricks

Missing dependencies

Frugalware's package manager - pacman-g2 - handles dependencies so that if you install package A and it requires packages B and C to work, all three packages are installed. Some applications have optional features and for these, the dependencies system doesn't apply. Take the example of a GStreamer-based media player as an example. If every one of the GStreamer plugins required to support every available audio format was installed, it might require a lot of disk space. Instead the person packaging the application may decide that only the most common audio formats will be specified as dependencies. When you try to play an unsupported audio file, it simply won't play. If you're in this situation the best tip is to start the application in a terminal and try playing or accessing the problem file. Hopefully you will get a message like "Plugin X could not be found." in the console, in which case you can then find and install the required package. If you're not sure just what the message means, or what to do about it, ask online via the forums or users' IRC channel.

Depending on people’s workload, the newsletter may be translated into French and Danish by their respective communities as soon as possible. Please be patient as it can take some time to translate each issue. Thanks to all those involved in providing and hosting these translations.

Release

To allow time for review and corrections, each newsletter is written ahead of its release date. Therefore it may not mention events which occurred in the few days before its release - e.g. security fixes. To be sure that you’ve got the very latest information on these topics, go to the appropriate page of the Frugalware web site.
